VMware Workstation에서 Rocky Linux 9.4 가상 머신을 생성하고 설치, 설정하는 과정을 정리하였다. <br>

---

### 1. 실습 환경
```text
- 가상화 프로그램: VMware Workstation Pro 25H2
- 설치 OS: Rocky Linux 9.4
- ISO 파일: Rocky-9.4-x86_64-minimal.iso
- 가상머신 이름: Rocky9-tem
- CPU: 1 Processor / 2 Core
- Memory: 2GB
- Disk: 50GB
- Network: NAT
```

<br>

### 2. Rocky Linux 가상 머신 생성
![이미지](/assets/images/rocky9-7/rockyinstall1.png) <br>
VMware Workstation을 실행한 후 왼쪽의 My Computer에 마우크 우클릭해서 `New Virtual Machine`을 선택하거나 File-New Virtual Machine...을 선택하여 새 가상머신을 생성한다.

<br>

- Custom 옵션 선택 <br>
![이미지](/assets/images/rocky9-7/rockyinstall2.png) <br>
가상 머신 설정을 직접 지정하기 위해 `Custom(advanced)`를 선택한다.<br>
Typical을 선택하면 기본값이 설정되지만, 실습에 맞게 CPU, 메모리 등을 설정하기 위해 `Custom(advanced)`를 선택하였다.

<br>

- 하드웨어 호환성 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall3.png) <br>
Hardware compatibility는 기본값인 `Workstation 25H2 or later`로 설정하였다.

<br>

- OS 설치 방식 선택 <br>
![이미지](/assets/images/rocky9-7/rockyinstall4.png) <br>
ISO 파일은 나중에 직접 연결하기 위해 `I will install the operating system later`를 선택하였다. <br>
이 옵션을 선택하면 먼저 빈 가상 머신을 생성한 뒤, 설정 화면에서 ISO 파일을 연결할 수 있다.

<br>

- Guest OS 선택 <br>
![이미지](/assets/images/rocky9-7/rockyinstall5.png) <br>
Guest operating system은 `Linux`를 선택하고, Version은 `Rocky Linux 64-bit`로 선택한다.

<br>

- 가상 머신 이름 및 저장 위치 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall6.png) <br>
가상 머신 이름은 `Rocky9-tem`으로 지정하였다. <br>
저장 위치는 실습용 VM을 관리하기 위해 별도의 폴더인 `D:\01_VMDK\Rocky9-tem`로 설정하였다. <br>
해당 가상 머신은 이후 여러 실습 환경에서 Clone하여 재사용할 템플릿 용도로 생성하였다. 

<br>

### 3. 가상 머신 하드웨어 설정
- CPU 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall7.png) <br>
Processor는 1개, Core는 2개로 설정하였다.

<br>

- 메모리 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall8.png) <br>
Rocky Linux Minimal 설치환경에서는 2048MB(2GB) 정도면 기본 실습을 진행하는 데 충분하기 때문에 2048MB(2GB)로 설정하였다.

<br>

- 네트워크 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall9.png) <br>
Network Type은 `NAT`로 선택하였다.

<br>

- 디스크 컨트롤러 및 디스크 타입 선택 <br>
![이미지](/assets/images/rocky9-7/rockyinstall10.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall11.png) <br>
I/O Controller Type은 기본 권장값인 `LSI Logic`을 선택하였고 Disk Type은 `SCSI`를 선택하였다.

<br>

- 새 가상 디스크 생성 <br>
![이미지](/assets/images/rocky9-7/rockyinstall12.png) <br>
새로운 Rocky Linux 설치를 위해 `Create a new virtual disk`를 선택하였다.

<br>

- 디스크 용량 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall13.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall14.png) <br>
디스크 용량은 50GB로 설정하였고 `Store virtual disk as a single file`을 선택해 가상 디스크 파일을 하나의 파일로 저장하도록 설정하였다.

<br>

- 불필요한 장치 제거 <br>
![이미지](/assets/images/rocky9-7/rockyinstall15.png) <br>
가상 머신 생성 전 `Customize Hardware`를 클릭하여 하드웨어 설정을 확인해 실습에 필요하지 않은 `Sound Card`는 제거했다.

<br>

### 4. Rocky Linux ISO 파일 연결
![이미지](/assets/images/rocky9-7/rockyinstall16.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall17.png) <br>
가상 머신 생성이 완료되면 `Edit virtual machine settings`를 클릭해 `CD/DVD(SATA)` - `Use ISO image file`을 선택하고 Rocky Linux ISO 파일을 연결한다.
이후 `Power on this virtual machine`을 클릭하여 가상 머신을 실행한다.

<br>

### 5. Rocky Linux 설치
- 설치 화면 <br>
![이미지](/assets/images/rocky9-7/rockyinstall18.png) <br>
가상 머신이 실행되면 위와 같은 설치 화면이 나온다. <br>
`Install Rocky Linux 9.4`를 선택하여 설치를 시작한다.

<br>

- 설치 언어 선택 <br>
![이미지](/assets/images/rocky9-7/rockyinstall19.png) <br>
설치 언어는 `한국어`로 선택하고 `계속 진행`을 클릭한다.

<br>

- 설치 목적지 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall20.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall21.png) <br>
설치 목적지에서 VMware에서 생성한 `50GiB` 디스크를 선택했다. <br>
저장소 구성은 기본값인 `자동 설정`으로 두고 `완료`를 클릭한다.

<br>

- KDUMP 설정 확인 <br>
![이미지](/assets/images/rocky9-7/rockyinstall22.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall23.png) <br>
KDUMP는 커널 충돌 시 원인 분석을 위해 사용되는 기능이다. <br>
실습 환경에서는 필수 기능이 아니므로 `kdump 활성화` 체크를 해제하고 `완료`를 클릭한다.

<br>

- root 비밀번호 설정 <br>
![이미지](/assets/images/rocky9-7/rockyinstall24.png) <br>
![이미지](/assets/images/rocky9-7/rockyinstall25.png) <br>
시스템 관리자 계정인 root의 비밀번호를 설정하였다. <br>
실습 편의를 위해 `root가 비밀번호로 SSH 로그인하도록 허용` 옵션도 체크하였다.

<br>

- 설치 시작 <br>
![이미지](/assets/images/rocky9-7/rockyinstall26.png) <br>
필수 설정이 완료되면 `설치 시작` 버튼이 활성화된다. <br>
버튼을 클릭하여 Rocky Linux 설치를 진행한다.

<br>

- 설치 완료 후 시스템 재시작 <br>
![이미지](/assets/images/rocky9-7/rockyinstall27.png) <br>
설치가 완료되면 `시스템 재시작` 버튼을 클릭하면 재부팅된다.

<br>

### 6. 설치 후 기본 확인
- root 계정으로 로그인 <br>
![이미지](/assets/images/rocky9-7/conf1.png) <br>
재부팅 후 로그인 화면에서 `root` 계정으로 로그인한다. <br>
정상적으로 로그인되면 `[root@localhost ~]#` 프롬프트가 표시된다.

<br>

- SELinux 상태 확인 <br>
![이미지](/assets/images/rocky9-7/conf2.png) <br>
![이미지](/assets/images/rocky9-7/conf3.png) <br>
`getenforce` 명령어로 SELinux 상태를 확인하였다. <br>
처음에는 `Enforcing` 상태였으며, 실습 편의를 위해 `setenforce 0` 명령어로 현재 실행 중인 SELinux를 임시로 완화하였다. <br>
이후 `grubby --update-kernel ALL --args selinux=0` 명령어를 사용하여 재부팅 후에도 SELinux가 활성화되지 않도록 설정하고, 재부팅 후 `Disabled` 상태를 확인하였다.
