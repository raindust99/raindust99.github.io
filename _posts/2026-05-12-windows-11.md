VMware Workstation에서 Windows 11 가상 머신을 생성하고 설치한 뒤, 실습 환경에 맞게 기본 설정을 진행하는 과정을 정리하였다. <br>

---

### 1. 실습 환경
```text
- 가상화 프로그램: VMware Workstation Pro 25H2
- 설치 OS: Windows 11 x64
- ISO 파일: ko-kr_windows_11_business_editi...
- 가상머신 이름: w11
- CPU: 1 Processor / 2 Core
- Memory: 4GB
- Disk: 64GB
- Network: NAT
- IP 주소: 10.0.0.102
- Subnet Mask: 255.255.255.0
- Gateway: 10.0.0.254
- DNS: 168.126.63.1 / 8.8.8.8
```

### 2. Windows 11 가상 머신

- 새 가상 머신 생성 <br>
![이미지](/assets/images/windows11/windows11install1.png) <br>
VMware Workstation을 실행한 후 File - New Virtual Machine...을 선택하여 새 가상머신을 생성한다.

<br>

- Custom 옵션 선택 <br>
![이미지](/assets/images/windows11/windows11install2.png) <br>
가상 머신 설정을 직접 지정하기 위해 Custom(advanced)를 선택한다. <br>
Typical을 선택하면 기본값이 설정되지만, 실습에 맞게 CPU, 메모리 등을 설정하기 위해 Custom(advanced)를 선택하였다.

<br>

- 하드웨어 호환성 설정 <br>
![이미지](/assets/images/windows11/windows11install3.png) <br>
Hardware compatibility는 기본값인 Workstation 25H2 or later로 설정하였다.

<br>

- OS 설치 방식 선택 <br>
![이미지](/assets/images/windows11/windows11install4.png) <br>
ISO 파일은 나중에 직접 연결하기 위해 I will install the operating system later를 선택하였다. <br>
이 옵션을 선택하면 먼저 빈 가상 머신을 생성한 뒤, 설정 화면에서 ISO 파일을 연결할 수 있다.

<br>

- Guest OS 선택 <br>
![이미지](/assets/images/windows11/windows11install5.png) <br>
Guest operating system은 Microsoft Windows를 선택하고, Version은 Windows 11 x64로 선택한다.

<br>

- 가상 머신 이름 및 저장 위치 설정 <br>
![이미지](/assets/images/windows11/windows11install6.png) <br>
가상 머신 이름은 w11로 지정하였다. <br>
저장 위치는 실습용 VM을 관리하기 위해 별도의 폴더인 D:\01_VMDK\w11로 설정하였다.

<br>

- 암호화 설정 <br>
![이미지](/assets/images/windows11/windows11install7.png) <br>
Windows 11은 TPM(Trusted Platform Module)이 필요하므로 가상 TPM을 지원하기 위한 암호화 설정 화면이 나타난다. <br>
Only the files needed to support a virtual TPM을 선택하고 암호를 입력한 뒤 Next를 클릭한다. <br>
해당 암호는 분실 시 가상 머신을 사용할 수 없으므로 안전한 곳에 보관해 두어야 한다.

<br>

- 펌웨어 타입 선택 <br>
![이미지](/assets/images/windows11/windows11install8.png) <br>
Windows 11은 UEFI 및 Secure Boot를 요구하므로 Firmware type은 UEFI를 선택하고 Secure Boot에 체크되어 있는지 확인한다.

<br>

### 3. 가상 머신 하드웨어 설정
- CPU 설정 <br>
![이미지](/assets/images/windows11/windows11install9.png) <br>
Processor는 1개, Core는 2개로 설정하였다.

<br>

- 메모리 설정 <br>
![이미지](/assets/images/windows11/windows11install10.png) <br>
Memory는 4096MB(4GB)로 설정하였다. <br>
Windows 11의 최소 요구 사양이 4GB이므로 권장값인 4GB를 할당하였다.

<br>

- 네트워크 설정 <br>
![이미지](/assets/images/windows11/windows11install11.png) <br>
Network Type은 NAT로 선택하였다.

<br>

- 디스크 컨트롤러 및 디스크 타입 선택 
    <div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 12px; align-items: start;">
    <img src="/assets/images/windows11/windows11install12.png" alt="이미지" style="width: 100%; height: auto; display: block;">
    <img src="/assets/images/windows11/windows11install13.png" alt="이미지" style="width: 100%; height: auto; display: block;">
    </div>
    I/O Controller Type은 기본 권장값인 `LSI Logic SAS`를 선택하였고 Disk Type도 기본 권장값인 `NVMe`를 선택하였다.
    <br>

<br>

- 새 가상 디스크 생성 <br>
![이미지](/assets/images/windows11/windows11install14.png) <br>
새로운 Windows 11 설치를 위해 Create a new virtual disk를 선택하였다.

<br>

- 디스크 용량 설정 <br>
    <div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 12px; align-items: start;">
    <img src="/assets/images/windows11/windows11install15.png" alt="이미지" style="width: 100%; height: auto; display: block;">
    <img src="/assets/images/windows11/windows11install16.png" alt="이미지" style="width: 100%; height: auto; display: block;">
    </div>
    디스크 용량은 64GB로 설정하였고 `Store virtual disk as a single file`을 선택해 가상 디스크 파일을 하나의 파일로 저장하도록 설정하였다.
    <br>

<br>

- 불필요한 장치 제거 <br>
![이미지](/assets/images/windows11/windows11install17.png) <br>
가상 머신 생성 전 Customize Hardware를 클릭하여 하드웨어 설정을 확인해 실습에 필요하지 않은 Sound Card는 제거했다.

<br>

가상 머신이 시작되면 Press any key to boot from CD or DVD.. 메시지가 나타나는데, 이때 아무 키나 눌러 ISO 부팅을 시작한다. <br>

    기본값을 한국어 환경으로 두고 다음을 클릭한 후 지금 설치를 클릭하여 Windows 설치를 시작한다.

<br>

- 설치 위치 선택 <br>
![이미지](/assets/images/windows11/windows11install26.png) <br>
Windows를 설치할 위치로 드라이브 0 할당되지 않은 공간을 선택하고 다음을 클릭하면 Windows 설치가 진행된다. <br>
디스크 크기는 앞에서 설정한 64GB로 표시된다.

### 9. Windows Defender 방화벽에서 ICMP 허용
실습 환경에서는 다른 PC 또는 서버에서 Windows 11 VM으로 Ping 테스트를 해야 할 수 있다. <br>
기본적으로 Windows Defender 방화벽에서 ICMP 요청이 차단될 수 있으므로, Ping 응답을 허용하기 위해 인바운드 규칙에서 ICMPv4 요청을 허용해야 한다.

<br>

- 제어판 실행 <br>
![이미지](/assets/images/windows11/windows11install50.png) <br>
Windows + R을 눌러 실행 창을 열고 control을 입력한 뒤 확인을 클릭하여 제어판을 실행한다.

<br>

- 보기 기준 변경 <br>
![이미지](/assets/images/windows11/windows11install51.png) <br>
제어판이 열리면 보기 기준을 작은 아이콘으로 변경한다.

<br>

- Windows Defender 방화벽 실행 <br>
![이미지](/assets/images/windows11/windows11install52.png) <br>
목록에서 Windows Defender 방화벽을 클릭한다.

<br>

- 고급 설정 열기 <br>
![이미지](/assets/images/windows11/windows11install53.png) <br>
왼쪽 메뉴에서 고급 설정을 클릭한다.

<br>

- ICMPv4 인바운드 규칙 찾기 <br>
![이미지](/assets/images/windows11/windows11install54.png) <br>
고급 보안이 포함된 Windows Defender 방화벽 창에서 인바운드 규칙을 선택하고 목록에서 파일 및 프린터 공유(에코 요청 - ICMPv4-In) 규칙을 찾는다.

<br>

- ICMPv4 규칙 사용 설정 <br>
![이미지](/assets/images/windows11/windows11install55.png) <br>
파일 및 프린터 공유(에코 요청 - ICMPv4-In) 규칙의 속성 창을 열고 사용함에 체크한 뒤 확인을 클릭한다. <br>
도메인과 개인 프로필 규칙 모두 해준다.

<br>

- Ping 허용 상태 확인 <br>
![이미지](/assets/images/windows11/windows11install56.png) <br>
사용 상태가 예로 변경되면 외부에서 해당 Windows 11 VM으로 Ping 요청을 보낼 수 있다.

### 10. 전원 옵션 설정
가상 머신이 일정 시간 후 절전 모드로 전환되면 실습 중 연결이 끊기거나 서비스가 중단될 수 있다. <br>
따라서 디스플레이 끄기와 절전 모드를 사용하지 않도록 설정한다.

<br>

- 전원 옵션 실행 <br>
![이미지](/assets/images/windows11/windows11install57.png) <br>
제어판의 모든 제어판 항목에서 전원 옵션을 클릭한다.

<br>

- 전원 관리 옵션 설정 변경 <br>
![이미지](/assets/images/windows11/windows11install58.png) <br>
사용 중인 전원 관리 옵션에서 설정 변경을 클릭한다.

<br>

- 절전 설정 해제 <br>
![이미지](/assets/images/windows11/windows11install59.png) <br>
디스플레이 끄기와 컴퓨터를 절전 모드로 설정을 모두 해당 없음으로 변경한 후 변경 내용 저장을 클릭한다.

### 11. Windows Update 서비스 설정
실습 환경에서는 Windows Update가 자동으로 실행되면서 재부팅이나 네트워크 사용량 증가가 발생할 수 있다. <br>
이를 방지하기 위해 Windows Update 서비스를 수동으로 제어할 수 있도록 설정하였다.

<br>

- 서비스 관리 도구 실행 <br>
![이미지](/assets/images/windows11/windows11install60.png) <br>
Windows + R을 눌러 실행 창을 열고 services.msc를 입력한 뒤 확인을 클릭한다.

<br>

- Windows Update 서비스 찾기 <br>
![이미지](/assets/images/windows11/windows11install61.png) <br>
서비스 목록에서 Windows Update를 찾는다.

<br>

- 시작 유형 변경 <br>
![이미지](/assets/images/windows11/windows11install62.png) <br>
Windows Update 속성 창에서 시작 유형을 사용 안 함으로 변경한 후 로그온 탭으로 이동한다.

<br>

- 로그온 계정 지정 <br>
![이미지](/assets/images/windows11/windows11install63.png) <br>
로그온 탭에서 계정 지정을 선택하고 찾아보기를 클릭한다. <br>
사용자 선택 창에서 고급 - 지금 찾기를 클릭한다. <br>
검색 결과에서 Guest 계정을 선택한 뒤 확인을 클릭한 후 암호와 암호 확인 값을 입력하고 다시 확인을 클릭한다.

<br>

### 12. 네트워크 IP 주소 설정
실습 환경에서 Windows 11 VM을 고정 IP로 사용하기 위해 네트워크 어댑터에 IP 주소를 수동으로 설정한다.

<br>

- 네트워크 연결 창 실행 <br>
![이미지](/assets/images/windows11/windows11install64.png) <br>
Windows + R을 눌러 실행 창을 열고 ncpa.cpl을 입력한 뒤 확인을 클릭한다.

<br>

- Ethernet0 IPv4에서 고정 IP 주소 입력 <br>
![이미지](/assets/images/windows11/windows11install65.png) <br>
네트워크 연결 창이 열리면 Ethernet0을 선택하고 속성을 클릭한 후 인터넷 프로토콜 버전 4(TCP/IPv4)를 선택, 속성을 클릭한다. <br>
IPv4 설정은 다음과 같이 입력하였다. <br>

    ```text
    IP 주소: 10.0.0.102
    서브넷 마스크: 255.255.255.0
    기본 게이트웨이: 10.0.0.254
    기본 설정 DNS 서버: 168.126.63.1
    보조 DNS 서버: 8.8.8.8
    ```
    
    설정이 완료되면 `확인`을 클릭하여 네트워크 설정을 저장하고 시스템을 재부팅한다.
